# example - remix (React Router v7)

React Router v7 (the framework formerly known as Remix) **resource route**: a
loader-only route module whose return value is a raw PDF `Response`.

## What's shown

- **`app/routes/invoice.pdf.tsx`** - the entire runtime glue: a `loader` that
  returns `pdf(byId('invoice')!.render())`. `pdf()` (Node entry,
  `@imprint-pdf/react`) returns a `Response` with the correct
  `Content-Type: application/pdf` headers, so the loader returns it verbatim. No
  default export → React Router treats it as a resource route.
- **`app/routes.ts`** - explicit route config mapping the URL `/invoice.pdf` to
  the module (avoids `flatRoutes` and the `invoice[.]pdf.tsx` dot-escaping).
- **`app/root.tsx`**, **`app/routes/home.tsx`** - minimal document shell and an
  index page linking to the resource route with a plain `<a>` (a client-side
  `<Link>` would not trigger a real document request).

## Run

```bash
pnpm --filter @imprint-pdf/example-remix dev
# → http://localhost:5173  →  click "Download invoice.pdf"  (or GET /invoice.pdf)
```

## Category

**B** - server framework returning a `Response` from a route loader.

## DX notes

- **Glue LoC:** 4 (the `loader` in `invoice.pdf.tsx`). Everything else is
  framework scaffolding shared by any RR7 app.
- **Entry:** `@imprint-pdf/react` `pdf()` (Node runtime, `ssr: true`).
- **Friction:** 🟡 - needs a **codegen step before `tsc` passes**. The typed
  loader imports `./+types/invoice.pdf`, generated by `react-router typegen`
  into `.react-router/types/`. The `typecheck` script runs it first:
  `react-router typegen && tsc --noEmit`. `tsconfig.json` also needs
  `rootDirs: [".", "./.react-router/types"]` and the generated dir in `include`.
  (A no-arg, untyped loader could skip typegen and would be 🟢, but every real
  RR7 app uses the typed route modules.)

## Codegen

```bash
react-router typegen     # writes .react-router/types/** (run before tsc --noEmit)
```
