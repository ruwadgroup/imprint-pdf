# Changelog

Per-package changes are tracked under `packages/<name>/CHANGELOG.md` and
generated by [Changesets](https://github.com/changesets/changesets). This file
keeps a hand-maintained summary at the repo level.

For where the project is headed, see [`ROADMAP.md`](ROADMAP.md). For the contract
about what's covered by semver, see [`STABILITY.md`](STABILITY.md).

## Unreleased

### Public API

- **Removed** `<View>` and `<Text>` from `@imprint/react`. Use HTML elements
  directly (`<div>`, `<span>`, `<p>`, `<h1>`–`<h6>`, `<a>`, `<img>`, `<table>`,
  `<ul>`, `<ol>`, `<li>`, `<section>`, `<article>`, `<aside>`, `<header>`,
  `<footer>`, `<main>`, `<nav>`, `<figure>`, `<figcaption>`, `<blockquote>`,
  `<pre>`, `<code>`, `<strong>`, `<em>`, `<small>`, `<label>`). They emit the
  same `PdfNode` types and carry semantic role tags for PDF/UA.
- **Added** `<Watermark>` — drawn behind page content on every page.
- **Added** `<Bookmark>` — registers an entry in the PDF outline; doubles as a
  named destination so `<a href="#title">` resolves within the document.
- **Added** `<PageNumber>` and `<TotalPages>` — resolved at draw time.
- **Added** `<Header>` / `<Footer>` running components — re-laid out per page.

### Layout and rendering

- Two-pass page allocation so forward `#anchor` links can resolve to pages that
  haven't been drawn yet.
- Disabled Taffy's pixel rounding — fractional intrinsic widths are preserved
  so paragraph breaking decisions stay consistent across the layout/draw
  boundary.
- `flex-1` (and `flex-grow:1; flex-shrink:1; flex-basis:0%`) now produce
  exactly equal columns instead of decaying to content-based sizing.
- Layout-time text measurement inherits `fontFamily` from ancestor nodes, so
  layout and draw agree on glyph widths.
- Per-corner `border-radius`, CSS `transform` (rotate / translate / scale /
  skew / matrix) with proper `transform-origin` handling, `box-shadow`,
  `background-image`, `object-position`, `aspect-ratio`, `line-clamp`,
  `text-indent`, `overflow: hidden` clipping.

### Forms

- Full AcroForm widget set: `<TextField>`, `<Checkbox>`, `<RadioGroup>`,
  `<Dropdown>`, `<Button>`, `<Signature>`. Field-flag bits live as named
  constants (`FF_REQUIRED`, `FF_MULTILINE`, `FF_RADIO`, `FF_PUSHBUTTON`,
  `FF_COMBO`) referencing the spec sections that define them.

### Testing

- New `@imprint/e2e` package — end-to-end test harness with PDF text
  extraction (`pdfjs-dist`), structural inspection (`pdf-lib`), and visual
  snapshot infrastructure (`pdftoppm` + `pixelmatch`). Seeded with regression
  tests for the layout/draw parity issues fixed in this round.

### Tooling

- VSCode workspace settings updated to run Biome's `quickfix` and
  `organizeImports` actions on save.
- `examples/bun-server` now declares `bun-types` directly so `@types/bun`'s
  reference resolves under pnpm's strict hoisting.
